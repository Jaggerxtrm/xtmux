## Code Changes

### Picker/event-log/messaging core

- `bin/tmux-session-picker` gained a JSONL event subsystem (`log_event`, `event_log_file`, JSON escaping), `log emit/tail/query`, `message-send/list/ack`, and explicit telemetry wrappers for `git`, `bd`, and `gh`.
- Existing orchestration commands now emit best-effort events: `safe-send-pointer --yes` logs `message.sent`; `handoff` logs `handoff.created`/`handoff.sent`; monitors log start/done/timeout/kill; `audit` logs `audit.run`.
- The telemetry wrapper logs `telemetry.command.started` before forwarding and typed result events after command completion, preserving the underlying exit code.

### Agent state and runtime hooks

- `scripts/agent-state.sh` now writes `agent.state` JSONL events on state transitions, including pane/session/state and optional bead/task/prompt/parent metadata.
- `extensions/pi-agent-state.ts` now handles pi `turn_end` and `agent_end`, records a compact last assistant message, emits `agent.turn.done`, and posts a short log-backed message to the parent session when `@agent_parent_session` exists.

### Monitoring launcher

- `scripts/xtmux-monitor.sh` opens a monitoring tmux layout. Default panes: dashboard, monitor-list/audit, raw JSONL tail. Optional panes: unacked messages, `agent.turn.done`, and git/bd/gh telemetry. It supports flags for session name, intervals, log path, picker command, attach/no-attach, and replacing an existing monitor session.
- `install.sh` now symlinks the script as `~/.local/bin/xtmux-monitor` and marks it executable with the other installed scripts.

### Tests

- `test/contract.sh` now covers event log writes, log CLI filters/tail, message channel send/list/ack, pi extension turn-done publishing surface, telemetry classification and wrapper logging, and `xtmux-monitor.sh` help/syntax plus live tmux 3-pane layout smoke.

### Bootstrap/config generated by `xt init --yes`

- `.xtrm/` was generated with runtime config, hooks, default/active skills, MCP config templates, and registry metadata.
- `.claude/settings.json`, `.claude/skills`, `.mcp.json`, `.pi/mcp.json`, `.beads/config.yaml`, `.beads/hooks/pre-commit`, `AGENTS.md`, and `CLAUDE.md` were updated/created by xtrm initialization.
